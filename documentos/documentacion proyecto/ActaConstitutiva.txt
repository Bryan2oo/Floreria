2. PROPÓSITO Y JUSTIFICACIÓN DEL PROYECTO
================================================================

La Floristería J&A, al incorporar como parte de su ecosistema de
trabajo un sistema web, se propone encaminarse hacia la transformación
digital, la optimización de transacciones comerciales y la calidad en
la experiencia de usuario. Al integrar un entorno digital eficiente,
ágil y seguro, se espera elevar la productividad administrativa,
operativa y comercial del negocio. Un análisis meticuloso de los casos
de uso y requerimientos permitirá definir el diseño del sistema, su
arquitectura y pila tecnológica convergente. Se busca incorporar una
interfaz dinámica, apegada a principios de diseño que provea navegación
ágil y una experiencia de compra de calidad, fidelizando a los clientes
y usuarios mediante resultados medibles y un mejor servicio.


================================================================
3. DESCRIPCIÓN DEL PROYECTO Y ENTREGABLES
================================================================

El proyecto consiste en el diseño e implementación de un sistema que
incluye una aplicación web basada en el patrón de arquitectura
Modelo-Vista-Controlador (MVC), lo cual permite centralización y
automatización integral de los ciclos operativos, una gestión ordenada
y eficiente del código y el flujo controlado de subprocesos. Incluye
además una base de datos esquematizada teniendo en cuenta la naturaleza
del negocio. Al tratarse de un sistema en el que se desacopla la lógica
de negocio de la interfaz de usuario, se garantiza una operación
escalable y robusta.

El sistema plantea cubrir funcionalidades administrativas como la
gestión de productos, la gestión de clientes, el seguimiento y manejo
de pedidos; y funcionalidades del usuario cliente orientadas a la compra
y realización de pedidos de flores y productos asociados. La aplicación
automatizará el ciclo de venta desde la captación del cliente hasta la
entrega final del producto.

ENTREGABLES PRINCIPALES
------------------------
- Software Base: Aplicación web funcional desarrollada en PHP 8.x y
  Bootstrap 5.
- Base de Datos Relacional: Esquema en MySQL que incluye tablas de
  productos, clientes, pedidos, carrito, promociones, roles, facturas.
- Módulo de Reportes: Herramienta de generación de informes de ventas
  y stock exportables en PDF (utilizando la librería FPDF).
- Documentación Técnica: Manuales de usuario para administradores y
  clientes, además de la documentación de la arquitectura del sistema.

4. REQUERIMIENTOS DE ALTO NIVEL
================================================================

REQUERIMIENTOS DEL PRODUCTO
----------------------------

Requisitos Funcionales:
  RF-01: El sistema permitirá al usuario registrarse.
  RF-02: El sistema permitirá el ingreso por medio de correo y contraseña.
  RF-03: El sistema permitirá al usuario observar el catálogo.
  RF-04: El sistema permitirá al usuario realizar un pedido de uno o
          varios productos, pudiendo seleccionar la modalidad de pago
          y entrega.
  RF-06: El sistema permitirá a los usuarios buscar algún producto por
          el Id o el nombre.
  RF-07: El sistema deberá permitir a los usuarios clientes gestionar
          el carrito de compras.
  RF-08: El sistema permitirá al Administrador la gestión de productos.
  RF-09: El sistema debe permitir al Administrador visualizar y descargar
          reportes de Ventas parametrizados por Cliente, Periodo,
          Producto en un Periodo, montos de compra.
  RF-10: El sistema debe permitir al administrador la gestión de
          promociones.
  RF-11: El sistema debe permitir al administrador eliminar una compra.

Requisitos No Funcionales:
  RNF-01: El sistema cuenta con encriptamiento de contraseñas y
           estrategias de seguridad.
  RNF-02: El sistema deberá cargar el catálogo de productos y las
           ofertas en un tiempo máximo de 3 segundos con conexión
           a internet estable.
  RNF-03: La interfaz del sistema deberá ser completamente responsiva
           y adaptarse a dispositivos móviles, tablets y escritorio
           utilizando Bootstrap 5.
  RNF-04: El sistema deberá ser compatible con los navegadores Google
           Chrome, Mozilla Firefox, Microsoft Edge y Safari en sus
           versiones más recientes.
  RNF-05: La plataforma deberá mantener una disponibilidad del 99%
           durante el horario de operación comercial (8:00 am - 8:00 pm).
  RNF-06: Las búsquedas de productos por ID o nombre deberán devolver
           resultados en menos de 1 segundo.
  RNF-07: El sistema guardará consistencia con la Guía de Estilo y
           estándares de diseño de interfaces y navegación, así como
           adaptación responsiva a dispositivos móviles.
  RNF-08: El sistema deberá contar en su backend con una base de datos
           robusta, en el estándar SQL, que contemple almacenamiento
           adecuado y escalable para los diferentes tipos de datos
           propios de la naturaleza del negocio.

REQUERIMIENTOS DEL PROYECTO
-----------------------------
- Metodología de Trabajo: El desarrollo se regirá estrictamente por
  la metodología SCRUM, con reuniones de planificación de sprint y
  revisiones periódicas con el docente (Product Owner).
- Entorno de Desarrollo: El software debe ser compatible con servidores
  locales Apache (XAMPP/WAMP) y utilizar Git/GitHub para el control
  de versiones.
- Restricción Temporal: El proyecto debe ser entregado y validado a
  más tardar en julio 2026, coincidiendo con el cierre del ciclo
  académico.
- Calidad de Código: El código debe seguir las convenciones de la
  arquitectura MVC, manteniendo modelos, vistas y controladores en
  carpetas separadas para facilitar el mantenimiento por parte del
  equipo.

6. PREMISAS Y RESTRICCIONES
================================================================

PREMISAS
--------
- El equipo de desarrollo cuenta con conocimientos previos en PHP,
  MySQL y arquitectura MVC.
- Se dispone de acceso a los servidores necesarios para el desarrollo
  y pruebas del sistema.
- La Floristería J&A proporcionará la información necesaria sobre sus
  productos, categorías y procesos internos.
- Los usuarios finales (clientes y administradores) dispondrán de
  conexión a Internet y dispositivo compatible.
- El proyecto se desarrolla en el marco académico de la asignatura
  "GESTIÓN DE PROYECTOS DE SOFTWARE" bajo supervisión docente.
- Se utilizará la metodología SCRUM para la planificación y ejecución
  de los sprints.

RESTRICCIONES
-------------
- El proyecto deberá completarse dentro del período académico 2025,
  cumpliendo con los plazos definidos en el plan de sprints.
- El presupuesto es limitado al entorno académico; se priorizarán
  herramientas y recursos de código abierto o gratuitos.
- El sistema no integrará pasarelas de pago reales en la versión
  inicial; el proceso de pago será referencial.
- El equipo está conformado únicamente por los 7 integrantes asignados;
  se considerarán como integrantes externos al Product Owner y al User
  Expert, quienes proveerán información y son responsables de dar
  seguimiento, validación y retroalimentación constantes al proyecto.
- El sistema será desarrollado para entorno web con adaptación
  responsiva para entorno móvil.
- La base de datos se alojará en MySQL local (localhost) para el
  entorno de desarrollo.
- Las funcionalidades de envío de correos electrónicos de recuperación
  de contraseña dependerán de la configuración del servidor de
  producción.

11. REQUISITOS DE APROBACIÓN DEL PROYECTO
================================================================

1. Cumplimiento funcional: El sistema debe cumplir con el 100% de los
   requisitos funcionales esenciales (RF-01 a RF-11), incluyendo la
   gestión integral del catálogo de productos, el registro de clientes,
   la administración de inventario, el funcionamiento del carrito de
   compras y la generación de reportes PDF.

2. Calidad Arquitectónica y Técnica: Todas las funcionalidades deben
   estar implementadas bajo el patrón MVC utilizando PHP 8.x. Se
   verificará que la lógica de negocio esté correctamente desacoplada
   de la interfaz de usuario y que las contraseñas se almacenen
   mediante bcrypt (RNF-01).

3. Pruebas de aceptación: Al menos el 90% de los casos de prueba deben
   ejecutarse con éxito. No deberán existir defectos críticos que
   impidan el flujo completo de compra.

4. Rendimiento y Usabilidad:
   - Disponibilidad: Mínimo 99% durante el horario de operación (RNF-05).
   - Tiempo de respuesta: Carga del catálogo en menos de 3 segundos
     (RNF-02) y búsquedas procesadas en menos de 1 segundo (RNF-06).
   - Responsividad: Adaptabilidad total en móviles y escritorio
     mediante Bootstrap 5.3 (RNF-03).

5. Entrega Documental: Se debe entregar el repositorio en GitHub junto
   con la documentación técnica y funcional completa, que incluya:
   - Manual de usuario (Administrador y Cliente Final).
   - Documentación de la arquitectura MVC y Diccionario de Datos MySQL.
   - Guía de despliegue en servidor local (Apache/XAMPP).
   - Informe final de pruebas funcionales y resultados de calidad.

6. Aprobación y Firmas: El Docente deberá revisar y validar el
   cumplimiento de los requisitos académicos y técnicos, firmando el
   acta de aceptación final junto con el Líder de Proyecto.
