3. DESCRIPCIÓN DEL PRODUCTO
================================================================

El sistema Florería J&A es una aplicación web fundamentada en el modelo
vista controlador (MVC) cuyo propósito general es centralizar y
automatizar las operaciones, abarcando desde la gestión de inventarios
y la gestión de clientes, hasta el seguimiento en tiempo real de los
pedidos.

El producto de software final garantizará una operatividad escalable y
robusta, optimizando las operaciones comerciales y mejorando la
experiencia de usuarios mediante un entorno digital agradable, seguro
y productivo.

6. EXCLUSIONES
================================================================

- No se contempla el desarrollo de una aplicación móvil nativa en esta
  versión.
- No se integrarán pasarelas de pago reales; el proceso de pago será
  referencial.
- No se incluye soporte posterior a la entrega final en julio 2026.
- No se desarrollarán módulos adicionales fuera del alcance aprobado
  (RF-01 a RF-11).
- Las funcionalidades de envío de correos de recuperación dependerán
  de la configuración del servidor de producción (no garantizado en
  localhost).


================================================================
7. REQUERIMIENTOS DE ALTO NIVEL
================================================================

- El equipo de desarrollo cuenta con conocimientos previos en PHP, HTML,
  JavaScript, MySQL y arquitectura MVC, desarrollo y ramificación en
  entorno GitHub, y contenerización.
- Se dispone de acceso a los servidores necesarios para el desarrollo y
  pruebas del sistema.
- La Florería J&A proporcionará la información necesaria sobre sus
  productos, categorías y procesos internos.
- Los usuarios finales (clientes y administradores) dispondrán de
  conexión a Internet y dispositivo compatible.
- Se utilizará la metodología SCRUM para la planificación y ejecución
  de los sprints.


================================================================
8. RESTRICCIONES
================================================================

- El proyecto deberá completarse dentro del período académico, con
  entrega final a más tardar en julio 2026.
- El presupuesto es limitado al entorno académico, priorizando
  herramientas de código abierto (PHP, MySQL, VS Code).
- El sistema será desarrollado para entorno web; la base de datos se
  alojará en MySQL local (localhost) para el entorno de desarrollo.
- El código debe seguir estrictamente las convenciones de la
  arquitectura MVC (Modelos, Vistas y Controladores en carpetas
  separadas).

10. CRITERIO DE ACEPTACIÓN
================================================================

Los entregables del proyecto se aceptarán cuando:

- Cumplan el 100% de los requisitos funcionales esenciales descritos
  en el acta.
- Todas las funcionalidades estén implementadas bajo el patrón de
  arquitectura MVC, con validación de estándares de calidad
  arquitectónica.
- Existan constancias documentadas de funcionamiento y pruebas
  exhaustivas de uso.
- Al menos el 90% de los casos de prueba definidos por el Analista de
  Calidad se ejecuten con éxito.
- Las contraseñas almacenadas en la base de datos estén encriptadas.
- El sistema demuestre los tiempos de respuesta establecidos
  (carga < 3 s y búsqueda < 1 s).
- Se entregue el repositorio de código en GitHub junto con la
  documentación técnica y funcional completa.
- El Patrocinador firme el acta de aceptación final.
