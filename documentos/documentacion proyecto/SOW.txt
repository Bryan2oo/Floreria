3. PROPÓSITO DEL ACUERDO DE TRABAJO
================================================================

El sistema J&A Florería es una aplicación web basada en el modelo
View-Controller (MVC), que tiene como objetivo centralizar y
automatizar los ciclos operativos, enfocándose en la gestión de
inventarios, gestión de perfiles de clientes y el seguimiento a pedidos
en tiempo real. El producto final garantizará una operación escalable y
estable, optimizará las operaciones comerciales y mejorará la
experiencia del cliente.


================================================================
4. ALCANCE DEL PROYECTO
================================================================

El proyecto comprende el diseño, desarrollo e implementación de un
sistema web.

INCLUYE:
--------
- Registro de usuarios, inicio de sesión (Login) y gestión de
  recuperación de contraseñas.
- Gestión de inventarios, categorización, stock y promociones.
- Módulo de carrito de compras y de generación de pedidos.
- Generación de reportes de ventas y stock en formato PDF.
- Sistema de monitoreo del estado de los pedidos desde su creación
  hasta la entrega final.
- Documentación técnica y manuales de usuario.
- Diseño y construcción de la base de datos relacional en MySQL.
- Diseño arquitectónico del sistema: esquema/tecnologías que soporte
  construcción, actualización, gestión del almacenamiento, despliegue
  y mantenimiento del sitema.

NO INCLUYE:
-----------
- No se contempla el desarrollo de una aplicación móvil nativa en
  esta versión.
- No se integrarán pasarelas de pago reales; el proceso de pago será
  referencial.
- No se incluye soporte posterior a la entrega final en julio 2026.
- No se desarrollarán módulos adicionales fuera del alcance aprobado
  (RF-01 a RF-07).
- Las funcionalidades de envío de correos de recuperación dependerán
  de la configuración del servidor de producción (no garantizado en
  localhost).

7. REQUISITOS DEL CLIENTE
================================================================

Compromisos del cliente:
- Proporcionar información detallada sobre productos, categorías y
  procesos internos.
- Designar a Doris Flores como interlocutora principal para
  validaciones.
- Garantizar el acceso a dispositivos con conexión a Internet para
  las pruebas finales.

REQUISITOS FUNCIONALES
-----------------------

Módulo de Autenticación y Usuarios (Roles):
  RF-USR-01: El sistema permitirá el registro de nuevos clientes
             capturando: nombre completo, correo electrónico (único),
             teléfono, dirección de entrega por defecto y contraseña.
  RF-USR-02: El sistema permitirá el inicio de sesión mediante
             credenciales (correo electrónico y contraseña).
  RF-USR-03: El sistema bloqueará temporalmente la cuenta del usuario
             tras 5 intentos fallidos consecutivos de inicio de sesión.
  RF-USR-04: El sistema permitirá la recuperación de contraseña
             enviando un enlace temporal y seguro al correo registrado.
  RF-USR-05: El sistema manejará control de acceso basado en roles
             (RBAC): Administrador, Vendedor y Cliente, restringiendo
             el acceso a los módulos según el rol.

Módulo de Catálogo y Productos:
  RF-PROD-01: El sistema permitirá al Administrador gestionar (Crear,
              Leer, Actualizar, Eliminar/Desactivar) productos. Campos
              obligatorios: SKU, nombre, precio, stock actual,
              descripción, categoría, estado de promoción e imagen(es).
  RF-PROD-02: El sistema mostrará a los Clientes un catálogo paginado,
              destacando en una sección principal los productos con
              estado "En promoción".
  RF-PROD-03: El sistema contará con una barra de búsqueda que permita
              localizar productos por SKU, nombre o palabras clave en
              la descripción, además de filtros por categoría y rango
              de precio.

Módulo de Carrito y Ventas/Pedidos:
  RF-VEN-01: El sistema permitirá al Cliente gestionar su carrito:
             agregar productos, modificar la cantidad, ver el subtotal
             dinámico y eliminar productos individualmente o vaciar el
             carrito completo.
  RF-VEN-02: El sistema descontará temporalmente el stock de un
             producto cuando se agregue al carrito y confirmará la
             reducción definitiva al concretar el pago (checkout).
  RF-VEN-03: El sistema permitirá al Cliente finalizar la compra
             (checkout) seleccionando un método de entrega (Recogida
             en tienda o Envío a domicilio) y un método de pago.
  RF-VEN-04: El sistema permitirá al Administrador y Vendedor cambiar
             el estado de un pedido (Pendiente, En Preparación, Listo
             para Envío, Entregado, Cancelado).

Módulo de Proveedores y Reportes:
  RF-PROV-01: El sistema permitirá al Administrador registrar los
              datos de los proveedores (nombre de empresa, contacto,
              teléfono, productos que provee).
  RF-REP-01:  El sistema permitirá al Administrador generar un reporte
              exportable (PDF/Excel) de las ventas concretadas en un
              rango de fechas específico.

8. REQUISITOS NO FUNCIONALES
================================================================

Seguridad:
  RNF-SEG-01: Las contraseñas se almacenarán utilizando el algoritmo
              de hash bcrypt con un factor de costo mínimo de 10 o
              superior (Salt automático).
  RNF-SEG-02: El sistema invalidará automáticamente la sesión activa
              de cualquier usuario (Admin o Cliente) tras 30 minutos
              de inactividad.
  RNF-SEG-03: Todas las comunicaciones entre el cliente y el servidor
              deberán estar cifradas bajo el protocolo HTTPS
              (TLS 1.2 o superior).

Rendimiento y Escalabilidad:
  RNF-REN-01: El tiempo de carga de la página inicial (catálogo y
              ofertas) no deberá exceder los 3 segundos en conexiones
              de banda ancha estándar.
  RNF-REN-02: Las consultas de búsqueda y filtrado de productos en
              la base de datos deberán retornar y renderizar resultados
              en menos de 1 segundo.
  RNF-REN-03: El sistema deberá ser capaz de soportar hasta 150
              usuarios concurrentes navegando el catálogo sin una
              degradación perceptible en los tiempos de respuesta.

Usabilidad y Compatibilidad:
  RNF-USA-01: La interfaz de usuario será construida bajo el paradigma
              Mobile First y será 100% responsiva (Bootstrap 5),
              adaptándose a resoluciones de móvil (320px+), tablet y
              escritorio.
  RNF-USA-02: El sistema garantizará compatibilidad visual y funcional
              con las dos últimas versiones estables de los
              navegadores: Google Chrome, Mozilla Firefox, Microsoft
              Edge y Safari.

Disponibilidad y Confiabilidad:
  RNF-DIS-01: La plataforma deberá garantizar un Uptime del 99.5%
              durante el horario comercial pico (08:00 AM - 08:00 PM).
  RNF-DIS-02: El sistema de base de datos deberá ejecutar un respaldo
              completo (Backup) automatizado diariamente a las 03:00 AM
              (fuera del horario comercial).
